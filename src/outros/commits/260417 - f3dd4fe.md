## 11. Backend — novos endpoints e correções

- **Medições clínicas** — API completa para sessões e medições (`measurement.controller`)
- **Paciente clínico** — registro automático de `patient` ao admitir um idoso (`patient.controller`, `C_idosoDetalhes`)
- **Administração de medicamentos** — API para registrar administrações (`administracao.controller`)
- **Auditoria** — endpoint de consulta ao histórico (`auditoria.controller`)
- **Ativar/desativar usuário** — novo endpoint dedicado (`toggle-usuario-ativo`)
- **Upload de arquivos** — proxy interno para o R2 (`/api/proxy/r2-upload`), evitando expor a URL do servidor de arquivos no frontend
- **Migração de pacientes** — script para criar registros `patient` de idosos já existentes (`migrate-patients`)
- **Correção** no Controller de Usuário — `JSON.parse` defensivo para evitar quebra quando o body já vem como objeto
