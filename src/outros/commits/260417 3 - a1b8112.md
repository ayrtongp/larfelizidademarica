## 2. Login biométrico (WebAuthn / Passkey) — novo

Suporte a autenticação biométrica via WebAuthn. O usuário cadastra a biometria do dispositivo no perfil e habilita o login sem senha.

### Funcionamento

- **Registro** — usuário abre perfil → aba Biometria → clica "Adicionar biometria" → nomeia o dispositivo → navegador solicita autenticação biométrica (Touch ID, Face ID, Windows Hello) → credencial salva no MongoDB.
- **Login** — usuário digita o username na tela de login → clica "Entrar com biometria" → navegador exibe prompt biométrico → servidor verifica e emite JWT igual ao fluxo normal.
- **Challenge** — armazenado temporariamente no documento do usuário (`_webauthnChallenge`) com TTL de 5 minutos; não exige sessão de servidor.
- **Toggle** — biometria pode ser habilitada/desabilitada no perfil; só é possível habilitar após cadastrar ao menos um dispositivo.
- **Múltiplos dispositivos** — cada dispositivo é uma credencial separada com nome, tipo (`singleDevice`/`multiDevice`), data de cadastro e transports.

### Arquivos criados

| Arquivo | Descrição |
|---|---|
| `src/pages/api/Controller/C_biometria.ts` | Controller API — 7 endpoints WebAuthn |
| `src/components/Paginas/Configuracoes/Biometria.tsx` | Aba de perfil: listar, cadastrar, remover dispositivos; toggle de login biométrico |

### Endpoints — `C_biometria.ts`

| Método | `?tipo=` | Descrição |
|---|---|---|
| GET | `dados-biometria` | Retorna credenciais e status `biometriaHabilitada` |
| POST | `registrar-inicio` | Gera challenge de registro e retorna `PublicKeyCredentialCreationOptions` |
| POST | `registrar-finalizar` | Verifica resposta do browser e persiste credencial |
| POST | `autenticar-inicio` | Gera challenge de autenticação e retorna `PublicKeyCredentialRequestOptions` |
| POST | `autenticar-finalizar` | Verifica assinatura, atualiza counter, emite JWT |
| PUT | `toggle-biometria` | Habilita / desabilita login biométrico |
| DELETE | `remover-credencial` | Remove credencial por `credentialId` |

### Modelo de dados — campos adicionados à collection `usuario`

```
webauthnCredentials: [{
  id: string              // credential ID (base64url)
  publicKey: string       // chave pública (base64)
  counter: number         // contador de assinaturas (anti-replay)
  deviceType: string      // 'singleDevice' | 'multiDevice'
  backedUp: boolean
  transports: string[]
  registeredAt: string    // ISO date
  name: string            // nome dado pelo usuário
}]
biometriaHabilitada: boolean
_webauthnChallenge: { challenge: string, expiresAt: Date }  // temporário
```
