## 1. Módulo de Escala de Funcionários (novo)

Implementação completa do módulo de escala de equipes com suporte a múltiplos tipos de turno.

### Modelo de dados — `escala_equipes`

| Campo | Descrição |
|---|---|
| `nome`, `descricao`, `cor` | Identificação visual da equipe |
| `membros[]` | `{ funcionarioId, usuarioId, nome, cargo, dataReferencia? }` |
| `regra` | `{ tipo, diasSemana?, dataReferencia?, horarioEntrada, horarioSaida }` |
| `ativo` | Ativar/desativar sem deletar |

Tipos de turno suportados:
- `dias_semana` — dias fixos da semana (ex: Seg/Qua/Sex)
- `alternado` — ciclo de 2 dias (trabalha/folga)
- `12x36` — igual ao alternado com horário de 12h
- `24x48` — ciclo de 3 dias

### `src/utils/escalaUtils.ts` (novo)

Lógica pura de cálculo, sem dependência de UI:
- `isWorkingDay(membro, regra, date)` — retorna se o funcionário trabalha em determinada data, respeitando a `dataReferencia` individual do membro sobre a da equipe
- `getWeekDays(referenceDate)` — retorna array de 7 datas a partir de domingo
- `getMonthDays(year, month)` — array de todas as datas do mês

### API — `C_escala.ts` + `C_escalaExcecoes.ts`

| Endpoint | Descrição |
|---|---|
| GET `getAll` | Todas as equipes com membros |
| GET `getById` | Equipe única |
| POST `new` | Criar equipe |
| PUT `update` | Atualizar dados/regra |
| PUT `updateMembros` | Full replace do array de membros |
| PUT `toggleAtivo` | Ativar / desativar |

`C_escalaExcecoes.ts` — exceções pontuais de folga ou plantão extra por funcionário/data.

### Componentes UI

- **`EquipeForm.tsx`** — modal em 3 seções (Info → Regra → Membros); cor com 8 chips predefinidos; campo `dataReferencia` por membro visível apenas em tipos cíclicos; inclui prestadores de serviço além de CLT
- **`EscalaGrade.tsx`** — grade semanal: linhas = funcionários, colunas = dias da semana; células coloridas pela cor da equipe; tooltip com horário ao passar o mouse; filtro por equipe; navegação semana anterior/próxima
- **`EscalaCalendario.tsx`** — calendário mensal com chips de iniciais dos funcionários por dia
- **`DiaDetalhePanel.tsx`** — painel lateral com lista completa de funcionários de plantão em um dia selecionado

### Página principal — `/portal/administrativo/escala`

- Abas "Semana" e "Mês"
- Botão "Gerenciar Equipes" abre modal com lista de equipes (editar, ativar/desativar, criar)
- Acesso RH via `PermissionWrapper groups={['rh']}`
- Link adicionado ao `rhMenu` no Sidebar

## 3. RH — Folha de Pagamento (nova)

### Modelo — `folha_pagamento`

Estrutura com lançamentos itemizados por funcionário (proventos e descontos individualizados):

```typescript
// src/types/T_folhaPagamento.ts
export interface T_Lancamento { descricao: string; valor: number; }

export interface T_FolhaPagamentoItem {
  funcionarioId: string; funcionarioNome: string; cargo: string;
  proventos: T_Lancamento[];   // ex: [{ descricao: 'Salário Base', valor: 3000 }, { descricao: 'DSR', valor: 200 }]
  descontos: T_Lancamento[];   // ex: [{ descricao: 'INSS', valor: 330 }, { descricao: 'Vale Transporte', valor: 100 }]
  totalProventos: number; totalDescontos: number; salarioLiquido: number;
}
```

### API — `C_folhaPagamento.ts`

- GET `getAll` — lista sem itens (só período + totais)
- GET `getById` — folha completa com itens
- POST `new` — cria folha (409 se período duplicado)
- PUT `update` — atualiza itens e recalcula totais
- PUT `anexarArquivo` — adiciona/substitui arquivo PDF da folha
- DELETE `delete`

Normalizador `normalizarItem()` garante que `totalProventos`/`totalDescontos`/`salarioLiquido` sempre sejam recalculados no servidor.

### `FolhaPagamentoForm.tsx` — modal criação

- Carrega funcionários ativos via `S_funcionariosCLT.getAtivos()`; pré-preenche `Salário Base` a partir do contrato
- **Modo Acordeão**: expande por funcionário para editar proventos e descontos; botões de atalho para os itens mais comuns (Salário Base, Horas Extras, INSS, IRRF, Vale Transporte…)
- **Modo Tabela**: toggle para `FolhaTabelaView`
- Totais gerais (bruto/descontos/líquido) em tempo real no rodapé

### `FolhaTabelaView.tsx` — componente compartilhado

Usado tanto no formulário de criação quanto na página de detalhe:
- `coletarColunas(itens)` — deriva colunas únicas a partir dos lançamentos de todos os funcionários (proventos primeiro, depois descontos)
- Cabeçalhos coloridos: verde para proventos, vermelho para descontos; linha separadora colorida abaixo
- Edição inline (inputs numéricos) quando `editando=true`; leitura quando `false`
- `+ P` / `+ D` no cabeçalho para adicionar nova coluna com autocomplete; `✕` no hover de cada coluna para removê-la de todos os funcionários
- Totais por coluna no `tfoot`

### Páginas

- **`/administrativo/folha-pagamento`** — lista de folhas com filtro por ano; ícone 📎 se tem arquivo anexo; botão "+ Nova Folha" (grupo RH)
- **`/administrativo/folha-pagamento/[id]`** — detalhe com toggle Acordeão/Tabela; modo edição dos valores; botão de anexar/substituir arquivo; resumo de totais no rodapé

## 7. Fix de segurança — Login.js

Crash ao acessar o app com token expirado ou ausente:

**Antes:** `const { userId } = jwt.decode(token)` — lançava exceção quando `decode()` retornava `null`.

**Depois:**
```js
export function getUserID() {
  if (typeof window !== 'undefined') {
    const token = localStorage.getItem('token');
    if (!token) return null;
    const decoded = jwt.decode(token);
    return decoded?.userId ?? null;
  }
  return null;
}

export function getUserFuncao() {
  const raw = localStorage.getItem('userInfo');
  if (!raw) return null;
  return JSON.parse(raw)?.funcao ?? null;
}
```

Mesma guarda aplicada a `getUserFuncao()`.
