## 1. Módulo de Listas de Compras — novo

Nova seção **Listas de Compras** dentro do menu Suprimentos (`/portal/suprimentos/listas`), desenvolvida para uso da nutricionista. Permite criar e gerenciar listas de mercado e sacolão com histórico completo.

### Funcionalidades

- **Histórico com tabs** — Mercado e Sacolão em abas separadas, com filtros por status e período
- **Cards no histórico** — exibem tipo, status, data, criado por, contagem de itens e barra de progresso de itens comprados
- **Criar lista** — modal com escolha de tipo, título, data prevista e observações
- **Usar como modelo** — duplica uma lista anterior com todos os itens copiados e checkboxes zerados; campo `baseadaEm` registra a origem
- **Gerenciamento de itens** — tabela com add/editar/remover inline; campos: nome, quantidade, unidade, categoria, preço estimado, observação
- **Save explícito** — barra amarela aparece quando há alterações não salvas nos itens; desaparece se o usuário desfaz todas as mudanças
- **Checkbox de comprado** — habilitado apenas no status `Finalizada`; desabilitado (com tooltip) em Rascunho e Comprada
- **Fluxo de status** — `Rascunho → Finalizada → Comprada` (terminal); `Finalizada` pode voltar para `Rascunho`; transições validadas no servidor
- **Auditoria** — todas as mutações (criar, duplicar, atualizar status, atualizar itens, atualizar info, excluir) registradas na collection `auditoria`
- **Soft delete** — listas excluídas recebem `ativo: false`, nunca removidas fisicamente
- **Tutorial integrado** — botão "Ajuda" na página de histórico abre modal com 7 passos navegáveis, tabela de status e fluxo visual

### Modelo de dados — collection `listas_compras`

```
tipo: 'mercado' | 'sacolao'
titulo: string
data: string (YYYY-MM-DD)
status: 'rascunho' | 'finalizada' | 'comprada'
itens: [{ _id, nome, quantidade, unidade, categoria, observacao, comprado, precoEstimado }]
criadoPor: string (userId)
criadoPorNome: string (denormalizado)
baseadaEm?: string (_id da lista de origem se duplicada)
ativo: boolean
```
