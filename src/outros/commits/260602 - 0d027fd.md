## 3. Migração de upload: GCS → Cloudflare R2

### `src/actions/DO_UploadFile.tsx` (reescrito)

| Função | Antes | Depois |
|---|---|---|
| `uploadArquivoPasta` | POST direto para Express `/upload` (GCS) sem auth | POST para `/api/proxy/r2-upload` com `Authorization: Bearer <token>` |
| `deleteArquivoPastaSubPasta` | DELETE Express `/delete?fileName&filePath` (GCS) | DELETE Express `/r2_delete?id=<r2FileId>` |
| `abrirArquivoR2` (nova) | — | Busca URL assinada via `/api/proxy/r2-url`; abre em nova aba |

Retorno atualizado: inclui `r2FileId` (= MongoDB `_id` do `arquivosr2`), `cloudURL: ''` (privado — sem URL permanente).

### Proxies Next.js

- **`/api/proxy/r2-upload`** (existia, agora em uso): verifica JWT; encaminha multipart ao Express `/r2_upload`
- **`/api/proxy/r2-url`** (novo): verifica JWT; chama Express `/r2_files/:id`; retorna `{ ok, url }` com URL assinada de 60s
- **`/api/proxy/rh-folha-zip`** (novo): verifica JWT; busca docs no MongoDB; obtém URLs assinadas; baixa arquivos; monta ZIP com `fflate`; envia buffer ao browser


### `.env.local`

```

```

### `src/pages/api/rh/folha-ponto-contabilidade.ts` (novo)

Endpoint `POST ?mes&ano` (requer JWT):

1. Busca docs `folha_ponto` do período no MongoDB
2. Resolve URLs assinadas via Express `/r2_files/:id`
3. Baixa cada arquivo
4. Monta ZIP com `fflate`
5. Faz upload do ZIP no R2 como **público** (via Express `/r2_upload`, `isPublic: true`) para obter URL CDN permanente
6. Chama `sendDocument(WPP_GRUPO_CONTABILIDADE, url, 'application/zip', filename, caption)`

> ZIPs ficam salvos em `rh/{ano}/{mes}/` no bucket público. Recomendado configurar lifecycle rule no R2 para expirar após ~30 dias.
