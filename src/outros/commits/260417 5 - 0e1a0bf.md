## 1. Biometria — discoverable credentials (login sem digitar username)

Refatoração do fluxo de autenticação biométrica para usar **passkeys descobríveis**: o browser apresenta automaticamente um seletor com as contas cadastradas, sem precisar informar o username antes.

### Mudanças no controller (`C_biometria.ts`)

- **Registro** — `residentKey` alterado de `'preferred'` para `'required'`, garantindo que a credencial seja armazenada como passkey no dispositivo
- **`autenticar-inicio`** — removida a exigência de `usuario` no body; `allowCredentials: []` instrui o browser a mostrar todas as passkeys disponíveis para o RP; challenge salvo na collection `webauthn_challenges` (TTL 5 min) em vez de no documento do usuário
- **`autenticar-finalizar`** — removido `userId` do body; o `userId` é derivado do `userHandle` devolvido pelo browser na resposta (`Buffer.from(userHandle, 'base64url').toString('utf8')`); challenge recuperado da collection temporária e deletado após uso

### Fluxo atualizado

```
Antes:  usuário digita username → clica "Entrar com biometria"
Depois: clica "Entrar com biometria" → browser abre seletor de passkeys → seleciona conta → loga
```

### Collection temporária — `webauthn_challenges`

```
_id:       challenge (string base64url — já é aleatório e único)
expiresAt: Date (5 minutos)
```
