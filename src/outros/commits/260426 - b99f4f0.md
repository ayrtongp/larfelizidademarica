## 2. Módulo Familiar — reestruturação arquitetural

### Problema resolvido

A versão anterior armazenava `pin_hash`, `id_residente` e `tipo: 'familia'` diretamente no documento `usuario`, tornando o vínculo 1:1 hardcoded e poluindo a coleção com campos que não existem para outros tipos de usuário.

### Nova arquitetura

**Coleção `familiar_residente`** (nova, N:N):
```ts
{
  usuario_id:   string,   // FK → usuario._id
  residente_id: string,   // FK → patient._id
  parentesco:   string,
  ativo:        boolean,
  createdAt, updatedAt
}
```

Familiar é agora um `usuario` comum com `funcoes: ['familiar']`. Nenhum campo especial no documento.

### Arquivos criados

| Arquivo | Descrição |
|---|---|
| `src/types/T_familiarResidente.ts` | Interface `T_FamiliarResidente`, `T_FamiliarResidentePopulado`, constante `PARENTESCO_OPTIONS` |
| `src/pages/api/Controller/C_familiarResidente.ts` | GET `getByResidente` (familiares de um residente com $lookup em usuario) · GET `getByUsuario` (residentes de um familiar com $lookup em patient) · POST `new` · PUT `toggleAtivo` · DELETE |
| `src/services/S_familiarResidente.ts` | `getByResidente`, `getByUsuario`, `createVinculo`, `toggleAtivo`, `removeVinculo` |
| `src/pages/familia/sem-acesso.tsx` | Página de fallback quando familiar não tem nenhum residente vinculado |

## 3. AI Studio — fix de timeout/413 no mobile

### Problema

Fotos tiradas com câmera de celular (≥ 4000×3000 px) geravam payloads de 3–5 MB em base64. O `express.json()` global do `nodejs.back` tinha o limite default de 100 kb e rejeitava o request com 413 antes que o middleware de rota (`limit: '10mb'`) entrasse em ação.

### Correção

- `nodejs.back/server.ts` — `express.json({ limit: '10mb' })` e `express.urlencoded({ limit: '10mb' })` globais
- `ai-studio/index.tsx` — função `compressDataUrl(dataUrl, maxPx=1200, quality=0.82)` chamada em `handleAnalyze()` antes de extrair o base64; escala a imagem para no máximo 1200px no lado maior antes de enviar

### Migração da chamada AI para DigitalOcean

`S_ai.ts` — `complete()` passa a chamar `${NEXT_PUBLIC_URLDO}/ai/complete` (fallback: URL hardcoded do app DigitalOcean) em vez do serverless `/api/Controller/C_ai`; contorna o limite de 10 s das funções Vercel para análises de imagem pesadas.

`fileToImage()` refatorado para comprimir via canvas (MAX_PX=800, JPEG_QUALITY=0.75) em vez de FileReader puro.

## 4. Fixes de data — "Invalid date format" no dashboard

### CalendarioM1 — aniversários

`getBirths()` em `portal/index.tsx` não filtrava residentes sem `data_nascimento`. Ao chamar `formatDateBR(undefined)`, retornava string inválida e o componente de calendário logava erros. Corrigido com:
- `.filter((item) => item.data_nascimento)` antes do map
- Guarda `if (!formatted) return null` e `if (!dia || !mes) return null` + `.filter(Boolean)` no resultado

### `datasImportantesToEventos()`

`T_datasImportantes.ts` — função refatorada para validar cada item antes de mapear: filtra `item.data && item.data.includes('-')`, verifica `parts.length >= 3`, retorna `null` para entradas inválidas e usa `.filter(Boolean)` no resultado.
